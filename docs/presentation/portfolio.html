Proof of depth

The hard problems, solved

Anyone can wire a template. These are the decisions that separate a demo from a system that survives contact with a real API budget, a real compliance rule, and a real phone.

data engineering

Real MLS data without a search API

MLS Grid only lets you filter on a few system fields — never county, price, or beds. The design replicates listings into a snapshot filtered in-app, and proxies photos on demand.

  • Budget-safe photo proxy — one call resolves a listing's photos, CDN-cached under the signed-URL window, paced under the rate cap, never stored.
  • Trailing-window rate limits respected — a learned constraint that once paused the account, now designed around.
automation

An engine that runs plans, not just builds them

The common gap in a CRM clone is that "auto-plans" never fire. This one enrolls, schedules, ticks on a cron, branches on conditions, and dispatches real sends.

  • TCPA-aware — a STOP footer and an opt-out suppression gate on every SMS.
  • Safe by default — a test mode that delivers nothing until deliberately turned on.
graphics · mobile

60fps 3D that converts on a phone

A real-time WebGL journey with a frame-rate governor, device-pixel scaling, portrait camera framing, and a boot fallback so the offer survives even when WebGL doesn't.

  • Resilient — a watchdog rescues old browsers and lost GL contexts into a static call-to-action.
AI orchestration

Shipping three apps with an agent loop

An orchestrator scopes work, parallel sub-agents build per app, every claim is verified against the live system before it's accepted, and a watchdog guards the process.

  • Force-multiplied — a solo operator moving at team throughput, with honesty gates so "done" means verified.
