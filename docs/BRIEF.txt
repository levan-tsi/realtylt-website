4) CONNECTION / CREDENTIALS AREA  *** OWNER FILLS THIS IN — DO NOT EMBED IN CODE ***
================================================================================
The builder must create a dedicated, secure config area (environment variables / secrets
manager), NOT hard-coded values. The OWNER inserts credentials himself in these slots:

  [MLS_API_KEY]        = <<< OWNER PASTES MLS/IDX API KEY HERE, IN THE SECRETS STORE >>>
  [MLS_API_ENDPOINT]   = <<< MLS/IDX feed base URL / provider >>>
  [MLS_FEED_ID/BOARD]  = <<< MLS board or feed identifier if required >>>
  [CRM_LEAD_WEBHOOK]   = <<< app.realtylt.com lead-intake endpoint / webhook URL >>>
  [CRM_API_TOKEN]      = <<< CRM auth token if required — OWNER inserts >>>

RULES FOR THE BUILDER:
- Read these ONLY from environment/secrets at runtime. Never write keys into source files,
  logs, client-side JS, HTML, or this document's committed copy.
- Provide a single config panel / .env template with the labeled slots above so the owner
  drops values in one place ("Connection area").
- For the autonomous test loop: use the owner-supplied TEST key from the secrets store to
  verify the IDX feed and lead flow, then the owner swaps in the PRODUCTION key at launch.
- After successful testing, the owner removes/rotates the test key. Do not persist it.

(Claude did NOT place any API key in this file. The key is a credential and must be entered
by the owner directly into the secrets store.)

5) INTEGRATIONS — DETAILED
================================================================================
A) MLS / IDX  (uses [MLS_API_KEY], [MLS_API_ENDPOINT])
   Powers: Search Listings, Featured Listings, New Listings, Top Areas county views,
   single-listing detail, saved searches, listing alerts, favorites.
   Components:
     - Search bar with filters (location, price, beds, baths, sqft, type).
     - Results grid + map view.
     - Listing card: price, address, beds|baths|sqft, "Listed With [brokerage]", View button.
     - Single-listing detail page (photos, description, features, agent contact CTA).
     - "Save this search" -> triggers listing-alert emails.
     - "Favorite" (heart) -> saved to user favorites.
   Test: with owner's test key, confirm live listings render and pagination works.

B) CRM LEAD CAPTURE  (uses [CRM_LEAD_WEBHOOK])
   Every form POSTs to app.realtylt.com. Payload: name, email, phone, message,
   interest-reason (see 7), source page, timestamp. Confirm lead appears in CRM/Agent Loop.

C) MORTGAGE CALCULATOR (no external service — pure JS)
   Inputs: Price, Annual Tax, Loan Term (yrs), Down Payment %, Interest Rate %, Monthly HOA,
   Monthly Insurance. Output: estimated monthly payment + breakdown
   (Principal / Taxes / HOA / Insurance with %). Include a Reset button.
